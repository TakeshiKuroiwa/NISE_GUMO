## 脆弱性の報告方法

**⚠️ セキュリティの脆弱性を GitHub の public Issue で報告しないでください。**

### 推奨される報告方法

1. **GitHub の Private Vulnerability Reporting を使用する（推奨）**  
   [Security タブ → Report a vulnerability](https://github.com/username/repo/security/advisories/new)

2. **メールで報告する**  
   security@example.com に送信してください。可能であれば PGP で暗号化してください。  
   PGP キー: [キーID またはリンク]

### 報告に含めてほしい情報

- 脆弱性の種類（例: XSS, SQLインジェクション, 認証バイパス など）
- 影響を受けるファイルのパス（フルパスまたは URL）
- 脆弱性を再現するための手順
- 概念実証コード（あれば）
- 影響の説明（攻撃者が何を達成できるか）

## 対応プロセス

| ステップ | 期間の目安 |
|----------|------------|
| 受信確認 | 48 時間以内 |
| 初期評価 | 5 営業日以内 |
| 修正リリース | 重大度に応じて 7〜30 日 |

修正後、報告者のクレジットを CHANGELOG に記載します（希望する場合）。

---

## 免責事項

善意の調査に基づく脆弱性報告は法的措置の対象になりません。ただし以下は含みません：

- 他のユーザーのデータへのアクセス
- サービスの妨害（DoS 攻撃）
- スパムの送信
