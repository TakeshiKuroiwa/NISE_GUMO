# 行動規範（Code of Conduct）
# ――あるいは「コードの道」

> *「あなたは誰ですか？」― ラマナ・マハリシ*  
> *「コントリビューターです」― 私たち全員*  
> *「本当に？」― ラマナ・マハリシ（再び）*

## 🌿 はじめに：そもそも「あなた」とは何か

このプロジェクトに参加するすべての人は、コントリビューター、メンテナー、Issue 荒らし予備軍を問わず、本質的に**純粋な意識**の表れです。

ラマナ・マハリシが言ったように、真の自己は傷つけられることも、deprecate されることもありません。しかし PR レビューのコメントは傷つきます。なのでガイドラインを作りました。

老子も言っています。*「千里の道も一歩から」* ――まず README を読むことから始めましょう。

---

## ☯️ 第一の道：無為（Wu Wei）のコミット

> *「為すことなくして、為されないことはない」― 老子*

良いコードとは、川が流れるように自然に書かれたコードです。  
**force push は川をせき止める行為です。** やめましょう。

- ✅ 流れに沿ってコードを書く
- ✅ レビューの流れに逆らわず、抵抗せず、でも主張はする
- ❌ `git push --force` を main に対して行うことは、宇宙の調和を乱します
- ❌ 「俺のやり方が正しい」という執着は、荘子の言う「小知」の罠です

---

## 🕉️ 第二の道：エゴの明け渡し（Issue 編）

パパジ（H.W.L. Poonja）はよく言いました。*「ただ静かにしていなさい」*  

Issue を開く前に、まず静かにしてください。5分間。  
本当に Issue が必要かどうか、自分の内側に問いかけてください。

**「今この瞬間に」存在する良い Issue の条件：**

1. 再現手順がある（「なんか壊れてます」は Issue ではなく詩です）
2. 期待する動作が書かれている
3. 怒りから書かれていない（一晩寝かせてから送信を）

> *「問題は常に今ここにある。解決策も然り。」― ムージ風*

---

## 🌊 第三の道：空（Emptiness）のプルリクエスト

空（くう）の概念によれば、最高の PR とは「変更が少ない PR」です。

荘子の料理人は牛を解体するとき、刃を骨と骨の間の**空間**に通しました。  
あなたの PR も同じように、既存のコードの隙間にそっと差し込んでください。  
3000行の差分は、骨ごと叩き割る斧です。

**PR の悟り度チェックリスト：**

- [ ] 1 PR = 1 つの目的（散漫な心は散漫なコードを生む）
- [ ] 変更行数は最小限（少ないほど道に近い）
- [ ] テストを書いた（無常に備えよ）
- [ ] セルフレビューした（自己探求の第一歩）

---

## 🙏 第四の道：コミュニケーションの純粋性

ムージはこう教えます。*「あなたの本性は愛そのものです」*  
レビューコメントにもその愛を込めてください。たとえ相手の変数名が `x2` でも。

### 許容される表現

- 「このアプローチについて別の観点から考えてみませんか？🙏」
- 「なぜここでこのパターンを選んだか教えてもらえますか？」
- 「素晴らしい実装です。一点だけ気になる箇所があります」
- 「私も同じ間違いをしたことがあります（昨日）」

### 許容されない表現

- 「なんでこんなコード書いたの？」（ラマナ・マハリシなら「なんで？と問う者は誰か？」と返しますが、それはレビューになりません）
- 「これは明らかに間違っている」（明らかなことは何もない、とムージは言う）
- 「前任者が書いたコードが最悪」（前任者もまた純粋な意識の表れです）
- 絵文字の過剰使用（🙏🙏🙏🙏は一つの🙏より弱い）

---

## 🌸 第五の道：執着からの解放（依存関係編）

> *「所有することなく、なすことなく、コントロールすることなく」― 老子*

npm パッケージへの執着は苦しみの源です。

- 必要最小限の依存を持つこと（`left-pad` 事件を覚えていますか）
- バージョンへの固執を手放すこと、ただし `package-lock.json` はコミットすること（これは矛盾ではなく、コアンです）
- `node_modules` を Git にコミットした者は、次の Issue で 108 回のレビューコメントを受けるでしょう

---

## 🌀 第六の道：無知の知（ドキュメント編）

老子は言いました。*「知る者は語らず、語る者は知らず」*  

ただしソフトウェアにおいては、**知る者も語るべきです。** README に書いてください。

- コメントのないコードは、解説のない公案（コアン）です――一部の人には美しいが、他の人には苦痛です
- 「自明である」と思ったことは、だいたい自明ではありません
- 未来の自分も他人です。親切にしてあげてください

## ⚡ 禁止事項（輪廻から抜け出せなくなるもの）

以下の行為は、このコミュニティおよびカルマの法則により禁じられています：

| 行為 | スピリチュアルな解釈 |
|------|------|
| ハラスメント・差別 | エゴの最も粗い表れ。即座に対処します |
| 他者の個人情報の無断公開 | プライバシーは聖なる空間です |
| 悪意あるコード（マルウェア等） | 意図は業（カルマ）として返ってきます |
| `main` への直接 push | 天道に逆らう行為 |
| レビューなしでのマージ | 一人の悟りは悟りではない |
| ミーティングで話し続けること | 沈黙もまたコミュニケーションです |

## 🌙 執行について

問題が発生した場合は [conduct@example.com](mailto:conduct@example.com) へ。

すべての報告は、判断なく、ただ**在るがままに**受け取られます。  
ただし適切な対処は行います。「無為」は「無視」ではありません。老子もそこは区別しています（たぶん）。

対応フロー：

```
問題発生
   ↓
深呼吸（全員）
   ↓
報告を静かに受け取る
   ↓
「これを経験しているのは誰か？」と問う（内部処理用）
   ↓
公正かつ迅速に対応する
   ↓
学びを得て、手放す
```
