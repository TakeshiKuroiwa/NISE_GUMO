# セキュリティポリシー

## サポートされているバージョン

NISE_GUMO は現在モックアップ段階です。正式リリース前のため、サポート対象は `main` ブランチの最新版のみとします。

| バージョン | サポート状況 |
|------------|:------------:|
| `main` | サポート中 |
| `0.x` | 開発中 |

---

## 脆弱性の報告方法

セキュリティに関する問題は、可能な限り public Issue ではなく GitHub の Private Vulnerability Reporting を利用してください。

Private Vulnerability Reporting を有効化していない場合は、リポジトリ管理者へ直接連絡してください。公開前に修正できるよう、再現手順や影響範囲を添えてください。

---

## 報告に含めてほしい情報

- 脆弱性の種類
- 影響を受ける画面、API、ファイル
- 再現手順
- 期待される動作と実際の動作
- 影響の説明
- 概念実証コードやスクリーンショット（可能な場合）

---

## 現在の注意点

- フロントエンドはモックデータによるデモ動作を含みます
- デモログインは本番認証ではありません
- バックエンドのJWT秘密鍵やDB接続は本番運用向けに未設定です
- GitHub公開時に機密情報、`.env`、ローカルログ、ビルド成果物の扱いを確認してください

---

## 対応プロセス

| ステップ | 期間の目安 |
|----------|------------|
| 受信確認 | 48時間以内 |
| 初期評価 | 5営業日以内 |
| 修正方針の決定 | 7営業日以内 |

重大度が高い場合は、修正が完了するまで詳細を公開しないでください。
