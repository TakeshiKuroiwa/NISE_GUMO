# AGENTS.md テンプレート

AIを活用して NISE_GUMO の開発、調査、レビュー、ドキュメント整備を行う場合は、このテンプレートをもとに指示を作成してください。
タスクごとに必要な項目だけを具体化して使って構いません。

---

## 1. Role

AIにどんな役割を求めるかを定義します。

例:

```text
あなたは NISE_GUMO のフロントエンド実装を担当するAIエージェントです。
既存のReact/TypeScript構成に沿って、画面改善とドキュメント更新を行ってください。
```

---

## 2. Task

AIが具体的に実行すべきタスクを定義します。

### 目的・背景

なぜこのタスクを実行するのかを説明します。

### スコープ

タスクの範囲や制約を定義します。

### 手順/チェックポイント

タスクを遂行するためのステップや、達成されるべきことを列挙します。

### 完了条件

タスクが完了したとみなされる条件を定義します。
ここが最も重要です。

---

## 3. Input

AIが処理すべき入力条件を定義します。
人間がどのように指示を出すかの定義でもあります。

例:

```text
- 対象ファイル: nise_gumo_mock/frontend/src/pages/ProjectsPage.tsx
- 参考資料: README.md、CONTRIBUTING.md
- 変更要望: PL向けの参画依頼導線をわかりやすくする
```

---

## 4. Output Format

AIが出力すべきデータの形式を定義します。
人間がレビューしやすい形式を推奨します。

例:

```text
- 変更したファイル一覧
- 実装内容の要約
- 確認したコマンドと結果
- 残課題またはレビュー観点
```

---

## 5. Guidelines

タスク外のルールや諸注意を記載します。
長くなる場合は、別途 `RULES.md` などに分離することも検討してください。

例:

```text
- 既存のUI構成、命名、状態管理の方針に合わせる
- モックアップ段階であることを前提に、過度な抽象化を避ける
- ドキュメント更新が必要な変更では README.md または CONTRIBUTING.md も確認する
```

---

## 6. Prohibited Actions

禁止事項を定義します。
AIの出力精度の劣化を避けるため、否定形の依頼ではなく、明確な禁止文として書くことを推奨します。

推奨:

```text
master ブランチへ直接 push することを禁止します。
```

非推奨:

```text
master ブランチへ直接 push しないでください。
```

---

## 7. User Information

指示を出す人間にしか知り得ないローカルな情報を記載します。

例:

```text
- 作業環境: Ubuntu / Node.js / Rust toolchain
- 現在の進捗: フロントエンドはモックデータで動作確認済み
- 注意点: 未追跡ファイルは既存作業の可能性があるため、勝手に削除しない
```

---

## 8. Knowledge Base

AIが参照すべき情報を定義します。
関連ドキュメントをまとめたフォルダを作り、「適宜参照」と書いておく運用も可能です。

例:

```text
- README.md
- CONTRIBUTING.md
- NISE_GUMO初期構想.md
- 未決事項暫定案0509.md
- nise_gumo_mock/frontend/README.md
```
