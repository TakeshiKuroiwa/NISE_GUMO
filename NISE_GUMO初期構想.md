# NISE_GUMO 初期構想ドキュメント

> 作成日: 2026-05-09  
> ステータス: 構想フェーズ

---

## 1. プロジェクト概要

| 項目 | 内容 |
|------|------|
| アプリ名 | NISE_GUMO |
| 目的 | カレンダーを用いたプロジェクト進捗管理 + コードエディタ機能の統合 |
| 対象ユーザー | エンジニア / プロジェクトリーダー / システム管理者 |

---

## 2. 開発環境

### ハードウェア

| 項目 | 内容 |
|------|------|
| 機種 | FUJITSU FMVA53A3R |
| CPU | Intel Core i7-6700HQ × 8スレッド |
| メモリ | 8.0 GiB |
| ストレージ | 1.0 TB |
| OS | Ubuntu 26.04 LTS (64-bit) / Wayland |

### 構築済み環境

- VSCode
- Git
- Node.js
- **Rust（rustup）**← 本プロジェクトで追加
  - `rustfmt` / `clippy` / `rust-analyzer` インストール済み

### Rustビルド設定

Rustのコンパイルはメモリ消費が大きいため、以下を設定：

```toml
# ~/.cargo/config.toml
[build]
jobs = 4
```

### VSCode拡張（Rust開発用）

- `rust-analyzer`（必須）
- `CodeLLDB`（デバッガ）
- `Even Better TOML`（Cargo.toml編集）

## 3. 技術スタック（決定事項）

### フロントエンド

| 用途 | 採用技術 |
|------|---------|
| フレームワーク | React + TypeScript |
| ガントチャート | `@dhtmlx/trial-react-gantt` 等（要最終選定） |
| コードエディタ | Monaco Editor（VSCode同エンジン） |
| 状態管理 | Zustand または TanStack Query |
| UI | shadcn/ui + Tailwind CSS |

### バックエンド

| 用途 | 採用技術 |
|------|---------|
| 言語 | Rust |
| Webフレームワーク | Axum |
| ORM / DB接続 | SQLx |
| 認証 | JWT（`jsonwebtoken` クレート） |
| リアルタイム通知 | WebSocket（`tokio-tungstenite`） |

### データベース

| 項目 | 内容 |
|------|------|
| DBMS | PostgreSQL |
| ホスティング | Supabase（無料枠：500MB / 2プロジェクト）|
| 接続方式 | SQLx + DATABASE_URL（.env管理） |

### 配布・パッケージング

| 項目 | 内容 |
|------|------|
| ツール | Tauri |
| 対象OS | Linux（.deb / .AppImage）/ Windows（.exe / .msi）/ macOS（.dmg） |
| 方針 | RustバックエンドとReactフロントエンドをTauriで統合してデスクトップアプリ化 |

## 4. DBテーブル設計（概要）

```
users          ─── ユーザー、ロール、プロフィール、設定、削除依頼フラグ
projects       ─── プロジェクト、メンバー、参画依頼ステータス
wbs_tasks      ─── タスク、担当者、期間、進捗ステータス
schedules      ─── 個人スケジュール（ユーザーが追加した予定）
notifications  ─── 通知内容、スコープ（全体/PJ/個人）、既読状態、返答選択肢
```

## 5. ユーザーロールと機能一覧

### 5-1. エンジニア

- ログイン（ID / パスワード）
- スケジュール画面（カレンダー + ガントチャート表示）
- 個人スケジュールの追加・編集
- コードエディタ起動（Monaco Editor）
- 新規プロジェクト参画依頼
- 参画中プロジェクト詳細確認
- プロフィール編集（アイコン / 表示名 / 一言メッセージ）
- 設定変更（レイアウト / ログイン非通知 / アカウント削除依頼）

### 5-2. プロジェクトリーダー

- エンジニアの全機能を継承（参画依頼・参画中詳細は非表示）
- プロジェクト管理画面
  - プロジェクト新規作成
  - WBS管理（詳細要件は未決）
  - 新規参画者招待
  - 参画者離脱処理

### 5-3. システム管理者

- 専用ログイン画面（管理用ID / パスワード）
- 新規アカウント追加
- アカウント削除（削除依頼があった場合のみ）
- プロジェクトリーダー権限譲渡
- 通知送信（スコープ：全体 / プロジェクト単位 / エンジニア単位）
  - 返答選択肢付き通知の作成も可能

## 6. UI方針

| 画面 | 方針 |
|------|------|
| 全体レイアウト | Discordライクなサイドバー構成 |
| スケジュール画面 | 月次カレンダー中央表示、過去・未来±1年ナビゲーション可能 |
| コードエディタ | VSCodeライク（左サイドバーにディレクトリツリー）、テーマ・フォント変更対応 |
| 管理画面 | シンプルなリスト管理 |
| 通知 | 画面上部にリアルタイムバナー表示 |

### プロフィール・設定（右上アイコンプルダウン）

- プロフィール編集：アイコン / 表示ネーム / 一言メッセージ
- 設定：画面レイアウト変更 / ログイン非通知オン / アカウント削除依頼
- ログアウト

## 8. 未決事項

| # | 項目 | 備考 |
|---|------|------|
| 1 | **WBS管理の詳細要件** | 現在は「メンバー個別カレンダー」として仮置き。要件を別途詰める |
| 2 | **ガントチャートライブラリの最終選定** | `@dhtmlx` / `react-gantt-chart` 等を比較評価が必要 |
| 3 | **コードエディタのファイル保存先** | ローカルファイルシステム（Tauri経由）か、クラウドストレージかを決定 |
| 4 | **複数人でのコード共有・同期方式** | リアルタイム共同編集が必要か、ファイル単位の保存で足りるか未定 |
| 5 | **ログイン非通知機能の実装方式** | WebSocketのプレゼンス管理と連携する設計が必要 |
| 6 | **プロジェクト離脱の権限設計** | エンジニアは離脱不可・PLのみ操作可能（実装ルールの詳細が必要） |
| 7 | **配布対象OS** | Linux必須。WindowsとmacOSの優先度を決定する |
| 8 | **Supabase無料枠の上限対応** | 将来的なストレージ増加時のセルフホスト移行計画を検討 |
